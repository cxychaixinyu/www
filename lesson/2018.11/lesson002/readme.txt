1、软件的分类
	两类:B/S 软件结构 和 C/S软件结构
		B/S:
		  B:browser  浏览器
		  S:server   服务器
		  
		特点：
			1、通用的客户端，客户端就是浏览器
			2、通用的协议，默认协议为 HTTP
			3、数据都存储在服务器端
		  
		C/S:
		  C:client   客户端
		  S:server   服务器
		
		特点：
			1、都有专门的客户端
			2、可以为任意的协议
			3、数据存储在服务器端和客户端
		
		B/S和C/S软件对比？
			b/s优点：
				1、所有的数据保存在服务器，相对安全
				2、不需要专门开发维护客户端，成本降低
			
			B/S缺点：
				1、所有的数据都存储在服务器端，服务器压力大

今日课程：
	1、URL：统一资源定位符
		完整的url地址：http://www.baidu.com:80/xxoo/ooxx/index.php?name=zhangsan&sex=nan#top
		
	
		1、http:// 协议 默认的协议
		2、www.baidu.com 域名
		3、:80 端口  默认的端口
		4、/xxoo/ooxx 所要请求的文件夹
		5、index.php  所要请求的文件
		6、？ 分隔符
		7、name=zhangsan&sex=nan  所携带的数据信息
		8、#top 片段标识符 --- 俗称锚点
	
	2、状态码（服务器）
		200 成功
		301 永久重定向
		304 没有修改
		403 拒绝访问
		404 找不到
		5XX 服务器错误
		
	3、HTML
		超文本标记语言：
			文本标记：通过标记以特定的形式进行展示
			超文本标记：可以标记图像，音频，视频等
	4、写HTML
		1、打开文件的扩展名
		2、使用编辑器打开
		3、设置编码格式---utf-8无BOM
	
	5、标签
		双标签和单标签
		
		双标签格式：
			<标签名 属性名1="属性值1" 属性名2="属性值2" ....>被标记的内容</标签名>
		
			1、标签可以拥有多个属性
			2、属性值使用单引号或者双引号，单双引号没有区别 
			
			<marquee></marquee>滚动标签
			
		单标签格式：
			<标签名 属性名1="属性值1" 属性名2="属性值2" ... />
			<br/> 换行
		
		实体字符标签：
			空格 &nbsp;
			<    &lt;
			>	 &gt;
			©	 &copy;	 
	6、html对回车和空格的处理
		1、多个空格当一个空格处理
		2、回车当空格处理（谷歌），火狐直接忽略回车，如果现有空格，之后紧邻回车则忽略空格
